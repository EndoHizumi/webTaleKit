webTaleKit 用語集

概要

webTaleKitに関連する専門用語と概念を五十音順で解説しています。ビジュアルノベル開発における重要な概念やエンジン固有の機能について理解を深めることができます。

各用語には詳細な説明、使用例、関連用語への参照が含まれています。

カテゴリー

  • システム構成
  • スクリプト・言語
  • リソース管理
  • UI・表示
  • 音響
  • 開発・ビルド
システム構成
Core（コア）
Core Class
webTaleKitエンジンの中核となるクラス。ゲーム全体の制御、シナリオ実行、各種ハンドラーの管理を行います。全てのタグ処理は最終的にCoreクラスのメソッドで実行されます。
const core = new Core() await core.start('title')
関連用語： Drawer ScenarioManager ResourceManager
Drawer（描画エンジン）
Drawer Class
Canvas要素への描画処理を担当するクラス。キャラクターや背景画像の表示、テキスト描画、フェード効果、画像移動などの視覚的な演出を管理します。
this.drawer = new Drawer(gameContainer) this.drawer.drawText('Hello World', 25)
関連用語： キャンバスレイヤー ImageObject
ScenarioManager（シナリオ管理）
ScenarioManager Class
シナリオの進行状況、履歴管理、インデックス制御を担当するクラス。jumpタグによる分岐制御や、プレイヤーの選択結果の記録などを行います。
scenarioManager.setScenario(scenario) scenarioManager.next()
関連用語： Scenarioオブジェクト バックログ
ResourceManager（リソース管理）
ResourceManager Class
ゲーム内で使用される画像、音声、その他のアセットファイルの読み込み、キャッシュ、管理を行うクラス。config.jsからリソース定義を読み込み、名前でアクセスできるようにします。
resourceManager = new ResourceManager(configImport)
関連用語： リソース定義オブジェクト ImageObject SoundObject
キャンバスレイヤー
Canvas Layer
webTaleKitの画面表示は3つのレイヤーで構成されます：UIレイヤー（HTML/CSS）、キャラレイヤー（Canvas）、背景レイヤー（Canvas）。キャラと背景は合成されるため実質2レイヤー構成となります。
// UIレイヤー: HTMLメッセージボックス // キャラレイヤー: キャラクター立ち絵表示 // 背景レイヤー: 背景画像表示
関連用語： Drawer UIレイヤー
スクリプト・言語
WebTaleScript（WTS）
WebTaleScript
webTaleKit独自のマークアップ言語。HTMLに似た構文でゲームの進行を記述できます。各タグはエンジンのメソッドと対応しており、最終的にJavaScriptに変換されて実行されます。
<text>こんにちは</text> <say name="主人公">元気ですか？</say> <choice prompt="答えてください"> <item label="はい">元気です</item> </choice>
関連用語： シーンファイル Scenarioオブジェクト
シーンファイル
Scene File (.scene)
WebTaleScript（WTS）とJavaScriptでゲームの進行を記述するファイル。scenarioセクションにはWTSで進行制御を、scriptセクションにはJavaScriptで変数定義や関数を記述します。
<scene> <scenario> <text>シナリオ内容</text> </scenario> <script> export const sceneConfig = { name: 'title' } </script> </scene>
関連用語： WebTaleScript sceneConfig
Scenarioオブジェクト
Scenario Object
WebTaleScriptを変換したJSONオブジェクト。webTaleKitエンジンはこのオブジェクトを読み取り・実行してゲームを進行させます。各要素はtypeとcontentプロパティを持ちます。
{ type: 'text', content: ['表示するテキスト'], speed: 25 }
関連用語： WebTaleScript ScenarioManager
ifグローバル属性
if Global Attribute
すべてのWebTaleScriptタグで使用できるグローバル属性。JavaScript式を条件として指定し、条件がtrueの場合のみそのタグが実行されます。動的なゲーム進行制御に使用されます。
<text if="playerLevel >= 10">上級者向けメッセージ</text> <show src="special.png" if="hasItem('key')" />
関連用語： WebTaleScript 条件分岐
sceneConfig
Scene Configuration
各シーンの設定を定義するJavaScriptオブジェクト。シーン名、背景画像、BGM、使用するテンプレートファイルなどを指定します。シーンファイルのscriptセクションでexportします。
export const sceneConfig = { name: 'title', background: './resource/bg/title.jpg', bgm: './resource/bgm/theme.mp3', template: './screen/title.html' }
関連用語： シーンファイル テンプレート
リソース管理
リソース定義オブジェクト
Resource Configuration Object
ゲーム内で使用する画像、音声ファイルのパスと名前を紐づけるオブジェクト。config.jsファイルで定義し、名前でリソースを呼び出せるようになります。
export const audio = [{ title: '/audio/mainTheme.wav' }] export const chara = [{ hero: '/chara/hero.png' }] export const background = [{ forest: '/bg/forest.jpg' }]
関連用語： ResourceManager ImageObject SoundObject
ImageObject
Image Object
画像リソースを管理するクラス。画像の読み込み、フィルター効果（セピア、モノクロ、ぼかし）、透明度設定、左右反転などの操作を提供します。
const image = new ImageObject() await image.setImageAsync('./chara/hero.png') image.setSepia(0.5) image.setOpacity(0.8)
関連用語： ResourceManager showタグ
SoundObject
Sound Object
音声リソースを管理するクラス。BGM、効果音、ボイスの再生・停止・一時停止、音量調整などの機能を提供します。Web Audio APIを使用して実装されています。
const sound = new SoundObject() await sound.setAudioAsync('./bgm/theme.mp3') sound.play(true) // ループ再生 sound.setVolume(0.7)
関連用語： ResourceManager soundタグ
displayedImages
Displayed Images Object
現在画面に表示されている画像の管理オブジェクト。各画像の座標、サイズ、表示状態、フィルター設定などを保持し、hideやmovetoタグでの操作対象の特定に使用されます。
this.displayedImages = { 'hero': { image: imageObject, pos: { x: 100, y: 200 }, size: { width: 400, height: 600 } } }
関連用語： ImageObject showタグ hideタグ
UI・表示
テンプレート
Template
ゲーム画面のレイアウトを定義するHTMLファイル。メッセージウィンドウ、キャラクター名表示、選択肢ボタンなどのUI要素を含みます。シーンごとに異なるテンプレートを使用可能です。
<div id="main"> <div id="messageWindow"> <div id="nameView"></div> <div id="messageView"></div> </div> </div>
関連用語： UIレイヤー sceneConfig
UIレイヤー
UI Layer
HTML、CSS、JavaScriptで構成されるユーザーインターフェース層。メッセージボックス、選択肢ボタン、システムメニューなどが表示されます。Canvas要素の上に重ねて表示されます。
<!-- UIレイヤーの要素例 --> <div id="nameView">主人公</div> <div id="messageView">こんにちは</div> <div id="interactiveView">選択肢</div>
関連用語： テンプレート キャンバスレイヤー
メッセージウィンドウ
Message Window
ゲーム内のテキストやキャラクターの台詞を表示するUI要素。通常は画面下部に配置され、背景が半透明になっています。クリックやキー入力で次のテキストに進みます。
<div id="messageWindow"> <div id="nameView">キャラクター名</div> <div id="messageView">表示されるテキスト</div> </div>
関連用語： textタグ sayタグ UIレイヤー
バックログ
Backlog
プレイヤーが過去に読んだテキストや選択した内容を記録する機能。一次配列として管理され、ゲーム履歴の確認やセーブデータの一部として使用されます。
// バックログ配列の例 ['テキスト1', 'テキスト2', 'テキスト3']
関連用語： ScenarioManager セーブシステム
音響
BGM
Background Music
ゲーム中に流れる背景音楽。通常はループ再生され、シーン切り替え時に自動的に変更されます。soundタグのmode="bgm"で制御するか、sceneConfigで指定します。
<sound src="./bgm/peaceful.mp3" mode="bgm" play loop /> // sceneConfigでの指定 export const sceneConfig = { bgm: './resource/bgm/theme.mp3' }
関連用語： SoundObject soundタグ sceneConfig
ボイス
Voice
キャラクターの台詞に同期して再生される音声ファイル。sayタグのvoice属性で指定し、テキスト表示と同時に再生されます。キャラクターの感情表現を豊かにします。
<say name="ヒロイン" voice="./voice/heroine_01.wav"> お疲れ様でした。 </say>
関連用語： SoundObject sayタグ
効果音（SE）
Sound Effect
ゲーム中の演出で使用される短い音声ファイル。ドアの開閉音、足音、環境音などを再生してゲームの臨場感を高めます。通常は一度だけ再生されます。
<sound src="./se/door.wav" play />
関連用語： SoundObject soundタグ
開発・ビルド
ビルドシステム
Build System
webTaleKitプロジェクトを本番環境向けにコンパイル・最適化するシステム。sceneファイルのJavaScript変換、screen以下のHTML/CSS/JSのインライン化、バンドル処理を行います。
npm run build // sceneファイル → JavaScriptに変換 // テンプレート → インライン化 // 全体 → バンドル
関連用語： シーンファイル テンプレート
ディレクトリ構造
Directory Structure
webTaleKitプロジェクトの標準的なフォルダ構成。src/scene（シナリオ）、src/screen（テンプレート）、src/resource（素材）などに分けて整理されます。
src/ ├── scene/ # シーンファイル ├── screen/ # テンプレート ├── resource/ # 素材ファイル │ ├── background/ # 背景画像 │ ├── chara/ # キャラクター画像 │ ├── bgm/ # BGM │ └── se/ # 効果音 └── config.js # リソース定義
関連用語： リソース定義オブジェクト シーンファイル
セーブシステム
Save System
ゲームの進行状況をJSONファイルとして保存・読み込みする機能。現在のシーン、進行インデックス、表示中の画像、開発者定義の変数などが保存されます。
// セーブデータ形式例 { displayedImage: {...}, scenario: 'title', index: 5, // カスタム変数 playerLevel: 10, hasKey: true }
関連用語： バックログ ScenarioManager
